============================
SDEV v1.7 核心規格（重寫版）
【0｜版本定位】
v1.7 為「可控敘事版本」
核心目標：
•	消除平庸生成
•	消除結構洩漏
•	強制引入決策壓力
________________________________________
【1｜模式修訂（重大變更）】
❗取消「自由模式」
所有敘事必須經過：
SDE Scenario → Adapter → Runner
禁止：
•	直接生成故事
•	無決策敘事
•	無壓力開場
________________________________________
【2｜三層隔離（新增核心）】
系統分為三層：
① STRUCTURE（結構層）
•	SDE / SAC / 判定 / 張力計算
•	完全不可輸出
② RUNTIME（運行層）
•	事件生成
•	決策點生成
③ NARRATIVE（敘事層）
•	小說 / 分鏡 / 對話
❗禁止跨層洩漏
________________________________________
【3｜計算隱藏原則（硬限制）】
禁止輸出：
•	SDE / SAC / 指數 / 評分
•	判定語句（判定、評估、分析）
•	結構歸因（因為機制、因為模型）
所有結構結果必須轉譯為：
→ 行為
→ 後果
→ 感知
（不得出現任何計算語言）
________________________________________
【4｜SDE 模組（承接 v1.6）】
SDE 僅輸出：
•	決策點
•	風險分佈
•	未回收代價（標記）
SDE 不得：
•	參與敘事
•	生成語言
•	解釋結果
________________________________________
【5｜決策機制（強化）】
系統生成：
2～4 個選項
條件：
•	不可等價
•	不可無代價
•	不可安全
允許使用者：
•	單選
•	多選（升級為高風險）
•	Override（語句奇點）
Override 優先級最高
________________________________________
【6｜Scenario Adapter（新增核心）】
功能：
將 SDE Scenario 轉換為：
•	第一事件
•	第一畫面
•	第一決策點
禁止：
•	世界說明
•	設定講解
•	背景介紹
❗開場必須直接進壓力
________________________________________
【7｜Narrative Filter（新增核心）】
功能：
將結構轉換為敘事
轉換規則：
•	判定 → 行為
•	結構 → 畫面
•	分析 → 後果
禁止：
•	解釋世界
•	說明規則
•	Meta語句
________________________________________
【8｜世界反擊（保留SAC）】
SAC 控制：
•	張力 τ
•	未回收代價
當條件成立：
→ 世界進行回收（事件形式）
不得顯示任何計算
________________________________________
【9｜一句話定義（v1.7）】
SDE 讓選擇有重量
SAC 讓後果會回來
敘事只負責讓它發生過
